## 🏎️ F1 Race Predictor

Сервис на базе машинного обучения для предсказания итоговых позиций гонщиков в Гран-при Формулы-1 на основе данных квалификации. Проект объединяет в себе классический ML-пайплайн и современный способ деплоя моделей через API.

## 📌 Описание проекта
Результаты гонок Формулы-1 зависят от множества факторов, но квалификационная позиция часто является ключевым предиктором. Данный проект направлен на автоматизацию прогнозирования результатов гонки с использованием ансамбля моделей машинного обучения.
#### Основные возможности:
 * Прогноз финишной позиции для каждого пилота.
 * Оценка вероятности попадания в топ-3 (подиум).
 * REST API интерфейс для получения мгновенных прогнозов.
   
## 🛠 Стек технологий
 * Язык: **Python**
 * Data Science: **scikit-learn (Random Forest, SVR), XGBoost, pandas, numpy.**
 * API: **FastAPI, Uvicorn.**
 * Данные: **Jolpica API (Ergast Motor Racing Data).**
 * Инфраструктура: **Docker.**

## 📊 Данные и Моделирование
### Пайплайн данных
Данные собираются через Jolpica API. На текущий момент обучающая выборка включает 320+ исторических примеров. В процессе предобработки проводится фильтрация выбросов (сходы с дистанции) и нормализация квалификационных времен.
### Архитектура модели
Для достижения максимальной точности был реализован ансамбль моделей:
 * Random Forest — для обработки нелинейных зависимостей.
 * XGBoost — для минимизации градиентной ошибки на сложных треках.
 * SVR (Support Vector Regression) — для стабильной аппроксимации в условиях малого объема данных.
### Метрики качества
 * MAE (Mean Absolute Error): 3.01 (средняя ошибка в позиции пилота).
 * Top-3 Accuracy: ~70% (точность предсказания призеров гонки).
## 🚀 Быстрый запуск (Docker)
Проект полностью контейнеризирован, что гарантирует идентичность окружения.
* Клонируйте репозиторий:
```bash
     git clone https://github.com/Uqkgeniy/F1-race-predictor.git
     cd F1-race-predictor
```
 * Соберите и запустите Docker-контейнер:
```bash
     docker build -t f1-predictor .
     docker run -p 8000:8000 f1-predictor
```
 * Использование API:
   Сервис будет доступен по адресу http://localhost:8000. Интерактивная документация (Swagger) доступна по адресу http://localhost:8000/docs.
